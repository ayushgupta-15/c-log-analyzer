# Contributing

Thanks for your interest in improving C Log Analyzer.

## Quick Start
- Fork the repo and create your branch from `main`.
- Run `make` to build.
- Make changes with clear commit messages.
- Open a pull request with a short description and rationale.

## Code Style
- Keep changes small and focused.
- Prefer simple, readable C (C99).
- Add or update documentation when behavior changes.

## Testing
- Run `make` before opening a PR.
- If you change parsing or reporting behavior, run `./loganalyzer logs/sample.log`.

## Reporting Issues
- Include steps to reproduce.
- Provide example log lines when possible.
